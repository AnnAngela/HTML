branch
list/create/delete



创建一个新分支.在feature文件夹下.
分支名为r1.0-test_branch

git branch -d <branch_name>
删除本地该分支

git branch -D <branch_name>
删除本地和远程该分支

git branch -r
列出远程分支


列出分支名带有r5.20的所有分支



列出本地和远程所有分支


—track : 切到远程分支.并追踪远程分支(仅需track一次)

如果第一次切远程分支没加—track.
那么就需要用
git branch --set-upstream-to=origin/<branch_name>
来重新追踪到远程分支




checkout
switch branches or restore working tree files

切换本地分支

未被暂存的文件.撤销修改.相当于discard.修改就没有记录了

未被暂存的文件.撤销所有修改

add
add file contents to the index

 
将所有的更改都stage起来


或者add指定路径下的文件


commit
record changes to the repository




push
update remote refs along with associated objects

如果你是创建的本地分支.
第一次push.就需要设置远程的跟踪分支.
以后就不用了



merge
Join two or more development histories together

将r1.0-test_branch分支合并到当前所在分支



rebase
reapply commits on top of another base tip




reset
reset current HEAD to the specified state

已被commit.用此命令可以回滚当前此次的commit
所有修改回到工作区
—hard: 不保留回滚的commit的修改
—soft/—mixed: 保留回滚的commit的修改
撤销未被暂存的见checkout命令


已被暂存的文件.修改会保存在工作区.修改还有记录.

revert
revert some existing commits


已被push.用此命令会把你revert的这次提交记录全部取消.然后commit一个干净的提交
版本会递增.不影响之前提交的内容

log



控制显示log的条数

显示当前分支最近的提交



remote


更改已存在的仓库域名
git remote set-url origin git@git.bilibili.co:<group>/<repo>
